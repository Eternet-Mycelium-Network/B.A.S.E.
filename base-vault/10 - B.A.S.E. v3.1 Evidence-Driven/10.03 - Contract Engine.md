# Contract Engine

> *Em vez de inferir componentes, defina contratos.*

## Conceito

Trocar:

```text
Isto é uma GPU (72%)
```

por:

```text
Este bloco requer: DMA, IRQ, MMIO, latência < 500ns
```

## Contratos de Requisito (extraídos do BIR)

```yaml
# contracts.yaml — extraído do Evidence DB + BIR
requires:
  - dma
  - interrupt
  - mmio_registers
  - max_latency: 500ns
  - gpio: 4
```

## Contratos de Provedor (catálogo de componentes)

```yaml
# component_contracts.yaml
RP2350:
  provides:
    - dma
    - interrupt
    - mmio_registers
    - spi
    - i2c
    - gpio: 48
  latency: 200ns
  constraints:
    package: QFN-56
    price: 1.50

W25Q128JV:
  provides:
    - qspi_flash
    - non_volatile_storage
  size: 16MB
  constraints:
    package: SOIC-8
    price: 0.80

PCM5102A:
  provides:
    - i2s_dac
    - audio_output
  constraints:
    package: TSSOP-16
    price: 1.50
```

## Matching Formal

```text
requires: dma, interrupt, mmio_registers, latency < 500ns
              ↓        ↓            ↓              ↓
provides:     ✅       ✅           ✅             ✅ (200ns < 500ns)
              └────────┴────────────┴──────────────┘
                        RP2350
```

## Tipos de Contrato

| Tipo | Exemplo | Verificação |
|------|---------|-------------|
| Presence | `requires: dma` | Componente tem DMA? |
| Numeric | `max_latency: 500ns` | Componente é mais rápido? |
| Count | `min_gpio: 16` | Componente tem GPIOs suficientes? |
| Interface | `requires: spi` | Componente tem SPI? |
| Package | `max_pins: 64` | Package compatível? |
| Budget | `max_cost: 5.00` | Cabe no orçamento? |
