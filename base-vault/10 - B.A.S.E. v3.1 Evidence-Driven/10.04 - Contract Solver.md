# Contract Solver

> *Matching formal entre `requires` e `provides`. Sem ML. Sem heurística. Sem magia.*

## Pipeline

```text
Contracts (requires)
       ↓
  ┌─────────────────┐
  │  Contract Solver │
  │  (formal match)  │
  └────────┬────────┘
           ↓
Component Catalog (provides)
           ↓
  ┌─────────────────┐
  │  Solution        │
  │  (satisfied?)    │
  └─────────────────┘
```

## Entrada

```yaml
requires:
  - dma
  - interrupt
  - mmio_registers
  - max_latency: 500ns
  - min_gpio: 16
```

## Saída

```yaml
solution:
  cpu: RP2350A
  memory: W25Q128JV
  audio: PCM5102A

  contracts:
    dma:              satisfied (RP2350: 8 channels)
    interrupt:        satisfied (RP2350: NVIC)
    mmio_registers:   satisfied (RP2350: 48KB SRAM mapped)
    max_latency:      satisfied (RP2350: 200ns < 500ns)
    min_gpio:         satisfied (RP2350: 48 ≥ 16)

  status: all_satisfied
```

## Algoritmo

```rust
fn solve(requires: &[Contract], catalog: &[Component]) -> Solution {
    for component in catalog {
        let mut results = Vec::new();
        for contract in requires {
            results.push(contract.verify(component));
        }
        if results.iter().all(|r| r.satisfied) {
            return Solution {
                component: component.clone(),
                results,
                status: Status::AllSatisfied,
            };
        }
    }
    Solution { status: Status::NoSolution }
}
```
