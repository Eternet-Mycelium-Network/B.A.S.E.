# Reference Design

> *Saída principal do pipeline. Um YAML descritivo — não uma PCB final.*

## Formato

```yaml
# reference_design.yaml
design:
  title: "LK Bootloader Reconstruction"
  version: 1
  source: "lk-sign.bin (disassembly)"

  architecture:
    cpu:
      part: RP2350A
      package: QFN-56
      price: 1.50

    memory:
      - type: qspi_flash
        part: W25Q128JV
        size: 16MB
        interface: spi
        price: 0.80

    audio:
      part: PCM5102A
      interface: i2s
      price: 1.50

  contracts:
    total: 5
    satisfied: 5
    violations: []

  bom:
    total_parts: 3
    estimated_cost: 3.80
    currency: USD

  pcb:
    type: engineering_draft
    layers: 2
    notes:
      - "Engineering draft — requires layout review"
      - "Power tree not included"
      - "Decoupling capacitors TBD"

  validation:
    status: pending
    contracts_verified: false
```

## Saídas

| Saída | Obrigatória? | Formato |
|-------|-------------|---------|
| `reference_design.yaml` | ✅ Sim | YAML |
| `reference_design.json` | ❌ Opcional | JSON |
| `contract_report.json` | ✅ Sim | JSON |
| `schematic.kicad_sch` | ❌ Opcional (engineering draft) | KiCad S-expr |
| `bom.csv` | ❌ Opcional | CSV |

## CLI

```bash
# Saída principal
base design spec.yaml -o output/
# → output/reference_design.yaml + output/contract_report.json

# Com PCB opcional
base design spec.yaml --pcb -o output/
# → + output/project.kicad_sch (marcado "Engineering Draft")
```

## O que o Contract Solver GARANTE

✅ **Contrato satisfeito** — o componente atende aos requisitos

O que ele NÃO garante:
❌ A PCB vai funcionar (depende de layout, alimentação, ruído, impedância)
❌ O firmware vai bootar (depende de implementação)
❌ A integridade elétrica (depende de layout)
