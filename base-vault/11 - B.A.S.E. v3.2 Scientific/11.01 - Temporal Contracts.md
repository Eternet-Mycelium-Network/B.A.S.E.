# Passo 9 — Temporal Contracts

> *Contratos de sequência: não apenas "tem DMA?", mas "write → dma_start → dma_complete → irq".*

## Problema

Hoje o `requires` só verifica **presença**:

```yaml
requires:
  - dma
  - irq
```

Mas o hardware real segue **sequências**:

```text
write(0x10000000, 1) → dma_start → dma_complete → irq(16)
```

## Solução

```yaml
requires:
  sequence:
    - mmio_write(0x10000000, 1)   # acorda GPU
    - dma_start                   # inicia transferência
    - dma_complete                # transf. terminou
    - irq(16)                     # sinaliza CPU

  max_total_latency: 5us          # janela total
  max_step_latency: 2us           # latência entre cada passo
  order: strict                   # ordem importa
```

## Implementação (Rust)

```rust
struct SequenceContract {
    name: String,
    steps: Vec<EventStep>,
    max_total_ns: u64,
    max_step_ns: u64,
    order: OrderConstraint,
}

struct EventStep {
    event_type: EventType,
    address: Option<u64>,
    value: Option<u64>,
    tolerance_ns: u64,
}

enum OrderConstraint { Strict, Relaxed, Any }
```

## Verificação

```rust
fn verify_sequence(trace: &[TraceEvent], contract: &SequenceContract) -> Vec<Violation> {
    for occurrence in find_patterns(trace, &contract.steps) {
        // 1. Eventos na ordem correta?
        if contract.order == Strict && !is_ordered(&occurrence) {
            violations.push(Violation::WrongOrder { .. });
        }
        // 2. Latência entre passos ≤ max_step_ns?
        for pair in occurrence.windows(2) {
            let dt = pair[1].timestamp_ns - pair[0].timestamp_ns;
            if dt > contract.max_step_ns {
                violations.push(Violation::StepLatency { .. });
            }
        }
        // 3. Latência total ≤ max_total_ns?
        let total = occurrence.last().timestamp_ns - occurrence.first().timestamp_ns;
        if total > contract.max_total_ns {
            violations.push(Violation::TotalLatency { .. });
        }
    }
}
```

## Exemplo: DMA do LK Bootloader

```yaml
contracts:
  - name: dma_transfer
    sequence:
      - mmio_write(0xa9bf0000, 0x01)
      - dma_start
      - dma_complete
      - irq(16)
    max_total_latency: 10us
    max_step_latency: 5us
    order: strict
```
