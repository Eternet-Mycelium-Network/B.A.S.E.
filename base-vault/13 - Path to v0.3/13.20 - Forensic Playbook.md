# Playbook forense — 1 página (v0.3)

**Objetivo da demo:** mostrar evidência → contratos → Reference Design no wedge UART, com engenheiro no loop.

## Setup (5 min)

```bash
git clone https://github.com/bmcc-DEV/B.A.S.E..git
cd B.A.S.E.
cargo build -p base-cli
./examples/pilot/run.sh          # regressão v0.2 + Capstone S1
# opcional: cargo test -p base-core --features solver_z3 --lib smt   # requer libz3
```

Artefatos: `examples/pilot/out/` · resumo `CASE_SUMMARY.md` · hashes `SHA256SUMS`.

## O que mostrar ao cliente (nessa ordem)

1. **Evidence** — `hardware_spec.yaml` + `evidence_db.yaml` (endereço UART `0x40034000`).
2. **Contratos** — replay feliz + fixture que **falha** (tensão honesta).
3. **Design** — MCU (ex. RP2040) + BOM estimada; contratos ≥70% no wedge.
4. **Limites** — PCB = `NOT FABRICABLE`; FW host ≠ silício; HIL = EXPERIMENTAL.

Script único: `./examples/pilot/run.sh` (também cobre o addendum Capstone v0.3).  
Case study: [[12 - Path to Real/12.20 - Pilot Case Study]].

## Não dizer

- “Substitui ASIC” / “PCB fabricável” / “prova Z3 always-on” / “HIL flasheou sozinho”.
