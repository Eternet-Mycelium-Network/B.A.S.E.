# SOW template — Port / diagnóstico industrial (v0.3)

> Copiar para o contrato do cliente. **Não** é SaaS turnkey. Humano no loop obrigatório.

## 1. Partes e foco

- **Cliente:** …
- **Prestador:** …
- **Alvo:** firmware / família de SoC / interface (ex.: UART / MMIO @ …)
- **Ferramenta de apoio:** B.A.S.E. (AGPLv3 ou licença comercial, conforme acordo)

## 2. Escopo (inclui)

| # | Entregável |
|---|------------|
| A | Análise assistida (`analyze` + Evidence DB) do blob acordado |
| B | Contratos temporais / replay quando traces existirem |
| C | Reference Design (MCU/BOM estimado) — draft de arquitetura |
| D | Relatório de viabilidade + riscos (Ψ / gaps / confiança) |
| E | Workshop de handoff (opcional, X horas) |

## 3. Não-escopo (exclui por default)

- PCB/Gerber **fabricável** ou DRC de fábrica
- Firmware de produção em silício / certificação
- Flash HIL automático sem probe + aprovação escrita
- Prova formal Z3 “universal” (apenas sob feature/`formal.yml` quando aplicável)
- Drop-in ASIC / garantia de pin-compatibilidade elétrico completa
- Turnkey SaaS / operação 24×7

## 4. Humano no loop

Toda decisão de arquitetura, BOM final, layout e release exige **aprovação de engenheiro nomeado** (Cliente ou Prestador). A ferramenta **acelera** evidência; não assina responsabilidade de produto.

## 5. Entregáveis e formato

- Pacote `out/` (YAML/JSON/MD) + hashes quando relevantes  
- Relatório PDF/MD de viabilidade  
- Vault/links para playbook [[13.20 - Forensic Playbook]] e case [[12 - Path to Real/12.20 - Pilot Case Study]]

## 6. Critérios de aceite

1. Smoke `./examples/pilot/run.sh` reproduzível no ambiente do Prestador (baseline).  
2. Bloco alvo do Cliente gera Evidence + ≥1 path de design **ou** relatório explícito de bloqueio.  
3. Linguagem do relatório **sem** claims da seção 3.  
4. Revisões: até N ciclos de feedback incluídos; além disso = change order.

## 7. Premissas / dependências

Cliente fornece firmware legal para análise, traces (se houver), e restrições de BOM/energia.

## 8. Preço e prazo (preencher)

- Fase 1 (diagnóstico): ___ / ___ semanas  
- Fase 2 (port assistido): sob change order  

## 9. Licença e IP

Código B.A.S.E.: conforme LICENSE/acordo comercial. Artefatos do estudo do Cliente: propriedade do Cliente, salvo libs upstream.
