# SOW Industrial Gate — promoção de claims

> Portão contratual. **Nenhum** claim industrial é REAL\* no código até este gate estar verde **e** o SOW do cliente assinado.
> Inspiração Paleo (§1.2 método científico; Axioma 2.4 minimizar Ψ): hipótese → evidência → falsificação → aceite — **não** “código/silício original único”.

Fonte Paleo (software): `PaleoComputação.pdf` (mapa conceitual; ≠ implementação B.A.S.E.).  
Mapa: [[22.31 - Paleo to BASE Map]].

## Ordem de promoção (menor risco → maior)

```text
1. HIL lab (flash assistido)  →  2. PCB sob engenharia  →  3. Auto-fix parcial
         ↑ bloqueio honesto se falta lab/probe/SOW/aceite
```

## Claim A — HIL lab / “production-assist”

| Pré-condição | Verde se | Bloqueio |
|--------------|----------|----------|
| A1 Probe | `ProbePresence::Detected` estável no lab do Cliente | Simulated only |
| A2 Programador | feature `hil_programmer` + `BASE_HIL_ALLOW_FLASH` + CMD documentado | flash no CI default |
| A3 SOP | procedimento escrito (operador, rollback, log) | “plug and flash” |
| A4 Receipt | `FlashReceipt.mode ≠ production` até revisão; depois mode tipado + auditoria | claim “production turnkey” |
| A5 Aceite | assinatura SOW §HIL | verbal only |

**Promove para:** *HIL lab assistido sob SOW* — ainda **≠** SaaS / CI flash automático.

## Claim B — PCB fabricável

| Pré-condição | Verde se | Bloqueio |
|--------------|----------|----------|
| B1 Draft base | sch/pcb com pins + `NOT FABRICABLE` removido **só** após B2–B5 | Gerber do draft atual |
| B2 Footprints | DB/footprints revisados por EE | template genérico |
| B3 DRC / netlist | DRC + netlist elétrico revisado | silkscreen-only |
| B4 Fab | 1ª placa montada / green-wire documentado | “fabricável” no README |
| B5 Aceite | EE + Cliente assinam release Gerber | tool sozinho |

**Promove para:** *PCB sob engenharia humana + tool* — entregável de **projeto**, não de CLI default.

## Claim C — Auto-fix parcial

| Pré-condição | Verde se | Bloqueio |
|--------------|----------|----------|
| C1 Evidência nova | traces/MMIO **novos** no loop (≠ só rename regs) | `reconstruct --continuous` sozinho |
| C2 Oráculo | goldens `diff` + regressão verde | sobrescrever `expected*/` |
| C3 Ψ / stop | report `stop_reason` + Ψ; `auto_fix_complete` só `true` se C1–C4 | marketing “conserta sozinho” |
| C4 Aceite | humano valida cada hipótese promovida | Specter sem review |
| C5 Escopo | SOW lista quais classes de bug (timing, name, classify) | “fix completa” |

**Promove para:** *auto-refine com evidência* — **nunca** “auto-fix completa universal”.

## Matriz rápida

## Falsificação (Paleo Estágio 2)

Antes de promover, o Prestador deve conseguir **falhar** o claim de propósito:

- HIL: negar flash sem Detected / sem ALLOW_FLASH  
- PCB: DRC falha ou EE rejeita  
- Fix: estagnação / golden diverge sem evidência nova  

Se não for falsificável, o claim **não** entra no SOW.

## Relação com código

Esta path (**v1.2**) **não** implementa A/B/C. Paths futuros (ex. Industrial HIL / PCB eng.) só abrem depois de:

1. Gate deste doc preenchido no SOW do Cliente  
2. Lab disponível  
3. Gates forenses + `run_study.sh` verdes
