# PaleoComputação → B.A.S.E. (mapa honesto)

> Fonte: *Anacroclastia e Paleocomputação Estrutural* (PDF, maio/2026) — guia **conceitual para software** (PaleoCLI).  
> B.A.S.E. opera em **hardware comportamental** (MMIO, contratos, Reference Design). O PDF **serve** como ética e vocabulário; **não** é o wedge de release.

## Correspondências

| Paleo (software) | B.A.S.E. (hardware) | Uso no Industrial Gate |
|------------------|---------------------|-------------------------|
| Binário = estrato / fóssil | Firmware + MMIO + traces = evidência | Sem evidência → sem claim |
| Compilador = erosão | Silício / SDK / otimização = perda de intenção | Expectativa ≠ “código original” |

| “Não há reconstrução perfeita única” | `auto_fix_complete: false` default | Claim C só parcial |
| Nethergate / Amiga / cases | Pesquisa | **Não** wedge industrial |

## Axiomas (tradução operacional)

1. **Conservação parcial** — algo do comportamento sobrevive no MMIO/traces; o resto é hipótese.  
2. **Erosão** — cada salto (binário→spec→PCB→silício) aumenta risco; claims exigem mais estrato.  
3. **Estratigrafia restringe** — SOW + lab + MCU conhecido > heurística universal.  
4. **Minimizar tensão** — Specter/reconstruct param em estagnação; não inventam evidência.

## O que NÃO importar do PDF

- Implementar PaleoCLI / PIR / HMC / cohomologia nesta path  
- Tratar o PDF como especificação de produto B.A.S.E.  
- Usar cases de jogos/PC como entregáveis de SOW industrial  

## Frase de ponte (comercial)

> *“Como a Paleocomputação trata o binário como fóssil (não como fonte perdida), o B.A.S.E. trata o firmware embutido como evidência comportamental: o Industrial Gate só promove PCB, HIL ou fix quando a hipótese sobrevive à falsificação no lab — não por marketing.”*
