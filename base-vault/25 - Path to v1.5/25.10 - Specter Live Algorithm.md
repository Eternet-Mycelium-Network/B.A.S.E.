# Specter Live — algoritmo de VM comportamental

> Fecha o loop: **emulador instrumentado → EvidenceDb → Ψ → hypothese/refine**.  
> Honesty: ≠ PCB fabricável · ≠ HIL production · ≠ OS turnkey (`generates_os=false`).

## Loop formal

```text
┌─────────────┐   MMIO/IRQ NDJSON     ┌──────────────────┐
│ QEMU (+TCG  │ ────────────────────► │ EvidenceDb live  │
│  plugin /   │                       └────────┬─────────┘
│  device gen)│◄──── QMP / gdbstub ────────────┤
└─────────────┘   (pause, inject, snapshot)    │
                                               ▼
                                     ┌──────────────────┐
                                     │ Ψ(B,H) + Twin    │
                                     └────────┬─────────┘
                                              │
                               OBSERVE → SCORE → REFINE
                                              │
                         paleo/phylo · mutar device · snapshot
```

### Passos

1. **Seed** — `HardwareSpec` (+ device QEMU Camada 9 ou `-machine virt` + firmware).
2. **Instrument** — plugin/device → eventos `MmioRead/Write`, `Irq` (schema `EvidenceDb`).
3. **Observe** — janela temporal → batch de evidência.
4. **Score** — `Ψ(Evidence, Spec)`; conclusiveness match / inconclusive / no-match.
5. **Refine** — ajustar hipótese / regenerar device / promover padrão paleo.
6. **Probe** (fase 3) — QMP stop/cont/savevm/inject IRQ.
7. **Halt** — estagnação / limiar; flags honesty sempre false.

## Formato de trace (MVP)

NDJSON, uma linha por evento:

```json
{"op":"mmio_write","addr":"0x10000000","value":"0x1","ts_ns":1000}
{"op":"mmio_read","addr":"0x10000004","ts_ns":2000}
{"op":"irq","vector":32,"polarity":"rising","ts_ns":3000}
```

Parser: `base_virt::trace::parse_ndjson_line` → `EvidenceEntry`.

## Trait `TraceSource`

```text
QemuNdjsonSource | FileNdjsonSource | (futuro) MameLuaSource | LibretroMemSource
```

Mesmo algoritmo; só muda a fonte.

## CLI (crate `base-virt`)

| Comando | Função |
|---------|--------|
| `base virt ingest` | NDJSON → `evidence_db.yaml` |
| `base virt score` | Evidence + Spec → Ψ (janela única) |
| `base virt run` | QEMU opt-in + ingest/score em janelas → `virt_session.yaml` |

## Artefactos

- `virt_session.yaml` / `.json` — Ψ por janela, honesty, qemu_exit
- `evidence_live.yaml` — EvidenceDb acumulado
- `CASE_SUMMARY_*_VIRT.md` — resumo piloto

## Honestidade

- Emular ≠ fabricar PCB ≠ port OS.
- Ψ live alta ≠ driver pronto / TaurOS.
- Checklist DTB 100% + QEMU boot ≠ SO completo.
- Produção HIL continua gated.
