# Twin↔guest

Replay de `EvidenceDb` (guest) contra `HardwareSpec` (twin/modelo):

1. Cada MMIO guest → **hit** se cai num bloco do Spec, senão **miss**.
2. Writes alimentam **shadow RAM** do twin (`addr → value`).
3. Reads podem devolver valor shadow se já escrito.
4. Blocos Spec sem evidência → `twin_only_blocks`.
5. `hit_rate` + Ψ em paralelo.

```bash
./target/debug/base virt twin \
  --spec examples/pilot_moto_g35/virt/hardware_spec_mame_stub.yaml \
  --evidence /tmp/ing_mame/evidence_db.yaml \
  -o /tmp/twin_mame
```

Demo alinhada: MAME sample `0xA00000` + `hardware_spec_mame_stub.yaml`.

Honesty: ≠ OS turnkey · shadow MMIO ≠ DigitalTwin BIR completo.
